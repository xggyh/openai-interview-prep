## 题目本质

设计 **News Aggregator System** (Google News / Apple News / Flipboard)：从全球新闻源抓取 → 去重 → 聚类同事件 → 个性化推荐 → 实时分发。

## 需求

- 100k+ 新闻源
- 1M+ articles/day
- 1B users
- 5-15 min freshness
- 个性化首页

## 整体架构

```ascii
   News sources (RSS / Sitemap / API)
       │
       ▼
  ┌──────────────┐
  │ Crawler/     │  poll sources every 5 min
  │ RSS Reader   │
  └──────┬───────┘
         │
         ▼
  ┌──────────────────┐
  │ Article          │  extract title / body / image / authoring
  │ Extractor        │  newspaper3k / readability
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────┐
  │ Dedup + Cluster  │  same-event articles grouped
  │ (embedding)      │
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────┐
  │ ML Tagging       │  topic / sentiment / quality score
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────┐
  │ Personalization  │  per-user feed ranking
  │ + Feed Service   │
  └──────────────────┘
         │
         ▼
   User app
```

## 核心组件

### 1. Crawler

每 source 一个 RSS feed 或 sitemap.xml。Polling 5-15 分钟（敏感时间 push subscription —— PubSubHubbub / WebSub）。

Crawl politeness：per-host rate limit (参考 [[cm80gligm049vtvyjklc6gxdw]])。

### 2. Article extraction

Raw HTML → 主文本：
- Readability algorithm（Mozilla 开源）
- 提取 title / body / publish_date / author / images
- 失败 fallback 到 OG tags

### 3. 去重 + 事件聚类

同事件不同 source 大量报道（"OpenAI launches GPT-5" 1000 个 article）。**聚类**：
- 计算 article embedding (sentence-BERT 384-dim)
- 实时聚类：新 article 找 cosine sim > 0.8 的已有 cluster
- 没命中 → new cluster (= new event)

每 cluster 是一"event"，UI 显示 representative article + "200 sources reporting"。

### 4. ML tagging

每 article 过 pipeline：
- Category classifier (tech / sports / politics / ...)
- Sentiment (positive / negative / neutral)
- Quality score (clickbait detect, factual signal)
- Entity extraction (people / orgs / locations)

存到 ES + 向量 store。

### 5. Personalization

User profile：
- Explicit interests (user picked)
- Implicit (click / read time / dwell)
- Embedding average of past clicks

Feed = candidate generation + ranking：
1. **Candidates**：recent (< 24h) articles + user's followed sources + topic match
2. **Ranking**：DNN takes (article features, user features) → click probability
3. **Diversity reranking**：不重复同 cluster

### 6. Real-time

新 article 进系统 5-15 min 后在 feed 出现。Push notification for breaking news:
- Editor's pick (高 quality + trending event)
- 或用户配置 alert keyword

### 7. Source quality

每 source 有 reputation score。Low-quality / fake news source → 降权 / blacklist。Editorial review for grey area。

## 取舍

| 决策 | 选择 | 替代 |
|---|---|---|
| Source ingestion | RSS / Sitemap / API | Generic crawl：noisy |
| Cluster | Embedding | TF-IDF：semantic 弱 |
| Personalization | DNN + collab filtering | Rule-based：static |
| Push | Selective alert | Every article：spam |
| Quality | Editorial + ML | None：fake news |

## 容量估算

- 1M articles/day × 50KB = 50 GB/day text + 0.5 GB embeddings
- Search QPS: 1B users × 5 feed queries/day = 60k QPS → ES + CDN cache hot feeds
- ML pipeline: 1M articles/day = 12 articles/sec → trivial

## 易错点

> [!pitfall]
> ❌ 抓不 dedup → user 看 100 篇同 event；
> ❌ 不 cluster events → ranking 把同事件多篇都推；
> ❌ 不做 quality control → fake news boost；
> ❌ Hot story 全 user 都看到 → echo chamber，加 diversity reranking；
> ❌ Push notification too aggressive → uninstall。

> [!key]
> 三大要点：(1) **Embedding-based event clustering** 处理同事件多 source；(2) **Multi-stage candidate + ranking + diversity**；(3) **Source quality + editorial layer** 防 fake news。

> [!followup]
> "如何处理 paywall articles？" → metadata + snippet only；"多语言？" → per-language pipeline + cross-language event linking；"Local news？" → geo-tag + user location；"如何 detect propaganda / state media？" → source labeling + ML credibility model。
