Leetcode 642. Design Search Autocomplete System

Coding Mid-level-Senior 原题 ↗
Google · Mid-level-Senior · 📋 2 · 🕒 Early November, 2025

Problem Statement

Design an online autocomplete system that processes characters one-by-one, returning the top 3 historically most frequent sentences that match the current prefix (ranked by frequency, then lexicographically), and inserts new sentences when the special character '#' is entered. The core challenge is supporting fast prefix queries and incremental updates over sentence frequencies, typically requiring a prefix-indexed structure (e.g., trie) or efficient top-k maintenance per prefix.

我的思路与解法

题目本质

LC 642 Design Search Autocomplete System：实现搜索框自动补全。

  • 初始给定一组历史搜索（sentence + hot 次数）
  • input(char)：用户每输入一个字符，返回该前缀下最热门 3 个 sentence（按 hot 降序，hot 相同时字典序升序）
  • 输入 #：表示用户完成输入，把当前 input 字符串加入历史（hot += 1）

经典Trie + Top-K。

数据结构

TrieNode:
  children: dict[char -> TrieNode]
  sentences: dict[full_sentence -> hot_count]   # 仅在以该 prefix 开头的 sentence 上记数

或者更省内存：每个 node 只记 "经过此 node 的 sentences"。每次查 top-3 sort 一下。

Python 实现

import heapq
from collections import defaultdict

class TrieNode:
    def __init__(self):
        self.children: dict[str, 'TrieNode'] = {}
        self.counts: dict[str, int] = {}   # sentence -> hot

class AutocompleteSystem:
    def __init__(self, sentences: list[str], times: list[int]):
        self.root = TrieNode()
        for s, t in zip(sentences, times):
            self._insert(s, t)
        self.cur_input = ""
        self.cur_node: TrieNode | None = self.root

    def _insert(self, sentence: str, count: int):
        node = self.root
        for ch in sentence:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
            node.counts[sentence] = node.counts.get(sentence, 0) + count
        # 注意：每个 prefix node 都记一次

    def input(self, c: str) -> list[str]:
        if c == "#":
            # finalize: add cur_input + 1 hot
            self._insert(self.cur_input, 1)
            self.cur_input = ""
            self.cur_node = self.root
            return []
        self.cur_input += c
        # advance trie pointer
        if self.cur_node is None or c not in self.cur_node.children:
            self.cur_node = None
            return []
        self.cur_node = self.cur_node.children[c]
        # Top-3 from current node's counts
        items = self.cur_node.counts.items()
        # 按 (-hot, sentence) 排序，取前 3
        top3 = heapq.nsmallest(3, items, key=lambda x: (-x[1], x[0]))
        return [s for s, _ in top3]

复杂度

  • __init__: O(总 sentence 字符数 × avg sentence length)（每字符一个 trie node + 每 prefix 都 store sentence）
  • input(c) 普通字符: O(K)，K = 当前 node 下 sentence 数（top-3 nsmallest = O(K log 3) ≈ O(K)）
  • input(#): O(L)，L = 输入长度

Space: 大 —— 每 prefix node 存所有该 prefix 的 sentence + count。1000 sentence × avg 长度 100 → 100k 总条目，可接受。

关键设计点

1. 在每个 prefix node 存 counts

node.counts[sentence] = hot —— 这样查 top-3 时直接看当前 node，不必往子树扫描。

代价：内存膨胀（每 sentence 出现在 trie 上 L 个 node）。但 input 操作变 O(K log 3) 很快。

替代：node 只存"经过此 node 的 sentence ID 集合"，每次查时扫子树聚合 top-3。空间省，但 input 变慢。

2. Top-K 排序 trick

按 (-hot, sentence) 排序：

  • -hot 降序就是 hot 升序的相反，min 取前 3 就是 hot 最大的前 3
  • 同 hot 时 sentence 字典序升序（min）

用 heapq.nsmallest(3, items, key=...) 一行完成。

3. Trie pointer 失效

如果用户输入的字符不在当前 node 的 children 里，整个 prefix 都不会匹配任何 sentence。cur_node = None，后续 input 直到 # 都返回 []。

4. # 重置

cur_input = "", cur_node = root。新一轮 input 开始。

易错点

易错点 / Red Flag

❌ 在 # 时插入 _insert(sentence, 1) 但没累加：要 += 1，已存在的 sentence 计数累加； ❌ Top-K 排序忘了 tie-break by sentence —— 多个 hot 相同时返回顺序不定； ❌ Trie pointer 失效后还想继续匹配 —— 直接 return []； ❌ 每个 input 都从 root 重新走 prefix —— O(L) per char，慢但能 AC；保持 cur_node 增量更新更优； ❌ 用 list 存 counts 而非 dict —— 重复 sentence 累加难。

核心要点

Autocomplete = Trie + top-K。Google 搜索框是这套加 ML ranking 的工业版。在 trie 上"每 prefix node 存子树聚合信息" 是空间换时间的经典 trade-off。

延伸 / Follow-up

"支持模糊匹配 / typo？" → BK-tree / Levenshtein automaton；"实时性能（百万 sentence）？" → top-K 不在 query 时算，离线预聚合 top-K 存到每 node；"如何 personalize？" → per-user trie 或全局 trie 但 user-aware ranking；"如何动态删除 sentence？" → trie 上递减 count，count==0 时清理 node。

because we don't want to reset the current prefix we are building if we've already processed at least one character. for example, if the input method is called with "i" then "a", when we process the "i" our current prefix should be updated to be "i" but when we process the "a" we want it to be "ia".
