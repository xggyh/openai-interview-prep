Design a Metrics Aggregator

System Design Senior рҹ“Ӣ 2 дәәжҠҘе‘Ҡ рҹ•’ жңҖиҝ‘пјҡLate December, 2025 еҺҹйўҳ вҶ—

Problem Statement

Design a metrics aggregation system that collects count-based metrics (like user signups, system errors, ad clicks) from services via a client library and displays them as histogram data on a dashboard. The system should support querying aggregated metrics within specific time frames for monitoring and analytics purposes.

1. Client SDK

class Counter:
    """Thread-safe counter that batches and flushes periodically."""

    _registry = {}
    _lock = threading.Lock()

    def __init__(self, name, **tags):
        self.name = name
        self.tags = tags
        self._value = 0
        self._lock = threading.Lock()
        with Counter._lock:
            Counter._registry[(name, frozenset(tags.items()))] = self

    def inc(self, amount=1):
        with self._lock:
            self._value += amount

    @classmethod
    def flush_all(cls):
        """е‘Ёжңҹз”ұ background thread и°ғз”ЁпјҢжҜҸз§’дёҖж¬Ў"""
        with cls._lock:
            batch = []
            for (name, tags), counter in cls._registry.items():
                with counter._lock:
                    val = counter._value
                    counter._value = 0
                if val > 0:
                    batch.append({'name': name, 'tags': dict(tags), 'value': val, 'ts': time.time()})
            send_to_ingest(batch)

def pick_resolution(start, end):
    span = end - start
    if span <= 1.hour: return '5s'
    if span <= 1.day:  return '1m'
    if span <= 30.days: return '1h'
    return '1d'

е®№йҮҸдј°з®—

  • 1M events/s Г— 100 еӯ—иҠӮ = 100 MB/s дёҠиЎҢ
  • еҒҮи®ҫ е№іеқҮ 100 unique series еӨ„зҗҶ (high cardinality дҫӢеӨ–) вҶ’ 5s жЎ¶ = 100 еҶҷ/5s/series = 20 QPS/series
  • 1 жңҲж•°жҚ®дҝқз•ҷ 5s зІ’еәҰ вҶ’ 100 series Г— 17280 buckets/day Г— 30 day Г— 16 B = ~830 MB пјҲеҚ• metricпјү

ж•ҙдҪ“з®—дёӢжқҘ 1 жңҲзІҫз»Ҷж•°жҚ® TB зә§пјҢ1 е№ҙиҒҡеҗҲдҝқз•ҷ < 1 TBгҖӮ

е…ій”®жҠҖжңҜз»ҶиҠӮ
